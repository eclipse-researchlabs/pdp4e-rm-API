Tasks & Project Schedule

Tasks in the Project Schedule

Beawre Risk Control always has a project schedule uploaded to the system. This project schedule is considered by the system to be the current valid project schedule and the one used to compute the risk status of all documents, once they have been linked to a task.

When we click the TASKS tab in the top of the screen, users can see a view that allows them to see information about the project schedule and the level of risk with respect to each task and linked documents.

Updating the project schedule

The project schedule can be updated at any time, by uploading a new version and replacing the previous one. This operation can be done at any time and as many times as necessary. The system also allows you to perform what-if analysis, i.e. check how changes in the project schedule may impact the risk status of current and future documents considered in the project schedule.

To update a project schedule or study the potential impact of changes, please press Upload Project Schedule +.

What-if analysis of project schedule changes impact

Before approving a new version of the project schedule, the owner of the project schedule or other users may be interested in understanding the effect of different changes in the project schedule on the risk status of the documents related to different tasks.

Beawre Risk Control allows users to upload a new file and perform an automated analysis of the differences between different project schedule versions.

When users press Upload Project Schedule + a new form will open called "Upload Project Schedule". This form shows a process with 7 different steps: (1) Waiting, (2) Uploading file, (3) Calculating risks, (4) Finding differences, (5) Revising changes, (6) Commiting changes and (7) Finished.

Here, we discuss the different steps related to what-if analysis:

  • Waiting: in this initial step, the system is waiting for the user to upload the new project schedule file to be inspected and compared with the current project schedule in the system. Users can drag and drop files from the file system browser (e.g. Explorer in Windows or Finder in Mac OS) or they can click in the box and a pop-up window will allow them to select the file to be uploaded to Beawre's system.
  • Uploading file: once the file has been selected, the system automatically moves to the next step and starts uploading the file. This step includes uploading the file to Beawre's server, inserting each element of the project schedule into the new database in production, and preparing the data for risk status calculation and comparison. Please note that this process can take from several seconds to several minutes, depending on the size of the project schedule.
  • Calculating risks: once the file has been uploaded and prepared for the operation, Beawre Risk Control will automatically move to the next step, where risks will be recalculated for all documents and tasks with the new project schedule file. Please note that this process can take from several seconds to several minutes, depending on the size of the project schedule and the documentation schedule.
  • Finding differences: once risk status has been calculated for each document and task, all the fields of each task are compared with the fields of the same task in the project schedule in the system. Risk status of the task will also be compared. Once the comparison is done, results of the comparison are displayed in the window for users to inspect.
  • Revising changes: once differences are found by the system, users can now visually explore all the differences between the previous and the new project schedules. Notably, they can also inspect changes in the risk status of tasks and understand what tasks are affected by the changes and what documents are impacted by the update.

We describe the remaining steps in the following subsection.

Please, note that this process can be cancelled at any time pressing Cancel upload process.

Confirming project schedule update

After inspection, users have the possibility to indicate to the system that the previous project schedule is to be replaced by the new one. In order to perform this action, please, press Commit changes button. Otherwise, if you prefer to keep the current project schedule, please press Cancel upload process button.

Search in the task list

The search mechanism for tasks in the project schedule are analogous to those used for documents in the documentation schedule, including a basic search capability and an advanced search.

Basic search mechanism

For a quick search, the basic search mechanism allows users to search any keyword or set of keywords across all the information related to a task. This search is useful for a quick exploration or when users look for some information but they are not sure about the specific field that contains that information. The mechanism is simple as users only need to type relevant words they are interested in and press enter or click the lens icon.

In order to see all the tasks again, users can press the X symbol to the right of the search bar.

Advanced search mechanism

There are situations where users may need to use very precise criteria to filter the tasks they are interested in or users may need to repeat certain queries very often. For those cases, the system provides an Advanced Search mode. To access to the advanced search please click on "Show Advanced Search".

Advanced search allows you to visually create any logic expression on the different attributes related to a task in the project schedule. A logic expression is composed of conditions on the data attributes (e.g. "Baseline start after a particular date", "Activity ID contains 4580" or "Only tasks linked to documents"), and a set of three operators (AND, OR, NOT) to combine multiple conditions.

In order to create multiple conditions, users can click on + Add rule. More than one rule need to be linked through an OR operator or an AND operator. With an OR operator the expression is true if at least one of the conditions is true. With an AND operator, the expression is true only if all the conditions are true. Users can select whether they want to use the OR or the AND operator by clicking the corresponding boxes. Users may also be interested in finding tasks for which the expression does not hold true. In those cases, they can click in the NOT box.

Different expressions with multiple conditions can be also combined to create any logical expression. To create a new expression with new conditions, click on + Add group.

In order to remove the filters on the whole task list, please click Reset button.

Please, note that depending on the complexity of a query, its execution may take up to several minutes.
