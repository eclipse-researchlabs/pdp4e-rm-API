Documents

Documentation Schedule

Beawre Risk Control helps you manage the documentation schedule of your project. In the Documents tab you can see all the documents in the documentation schedule. Please note that you can only see the documents you have access to (depending on your role you may not have privileges to see all the documents).

From the Documents tab you can list the documents in the documentation schedule, add new documents or search specific documents and filter them based on multiple criteria, etc.

Adding a new document

There are different mechanisms to create a new document in the documentation schedule: automatically and manually.

Automatic documentation addition

The easiest mechanism to create a new document in the documentation schedule is to upload the new document in Dymadoc. When Beawre Risk Control receives an e-mail from Dymadoc, it automatically adds the document to the documentation schedule. Note, while this option is transparent to the user, it does not allow Beawre Risk Control to track expected documents that have not been submitted by the subcontractor yet.

Manual documentation addition

In order to add a new document in the documentation schedule manually, please click on Add New Document. This option is important as it allows the Beawre Risk Control tool to know which documents are expected and to warn owners when documents are not yet submitted by subcontractors and approaching their deadlines.

When a document is created manually, Beawre Risk Control requires different information about the document. Our solution shows possible data options to complete the information about the document quickly. Please note that the suggested information is optional and users can type new information in any of the different fields. Also note that some fields are compulsory - marked with a red asterisk. The system will not allow users to create a new document if these fields are not complete.

Information about a document in the documentation schedule

For each document in the documentation schedule list, users can explore related information. The documents list shows:

  • the title and the id of each document
  • the information source from where the information about this document is extracted. There are two options:
    • from the former excel files used to manage the documentation schedule and
    • from Dymadoc
  • the field "Concerns", to indicate the content type of the document.
  • the field "Version", to indicate the most recent version of the document.
  • the risk status, to indicate the current level of risk of this document. In order to know more about how we calculate the level of risk, please check Section "Risk analysis of a document" below.

In additon, if you click in the + symbol appearing on the left of each document name and id, you will unfold further document information.

In particular, you can see:

  • All the existing versions of a particular document.
  • Statistics about a document. You can hover on each value to see the specific meaning of the statistics displayed in the user interface and, when relevant, information about how they have been calculated.
  • The full reference of the document.
  • For each version, the current status of the version. There are several steps a particular version of a document needs to go through. If a step has already taken place, there will be a check next to the step and the actual date when the step happened will be shown. In some cases, this information will not be available and the system will indicate this displaying "Not available". If the step has not taken place yet, the date will not be shown, and there will be a number indicating the position in the process of the next expected step.
  • The deadline for the document. This information can only be seen if the document was linked to a task in the project schedule.

Search in the documentation schedule

In order to look for a particular document or set of documents, the system provides two different mechanisms for searching: a basic search mechanism and an advanced one.

Please note that the search function is one of the essential features of this system. Searching for specific documents does not only constrain the list of documents visible in the document schedule, but it also affects the data taken into consideration in the ANALYTICS tab. This means that by executing different search queries, it is possible to visually explore different subsets of data through the plots represented in the analytics dashboard.

Basic search mechanism

For a quick search, the basic search mechanism allows users to search any keyword or set of keywords across all the information related to a document. This search is useful for a quick exploration or when users look for some information but they are not sure about the specific field that contains that information. The mechanism is simple as users only need to type relevant words they are interested in and press enter or click in the lens icon.

In order to see all the documents again, users can press the X symbol on the right of the search bar.

Advanced search mechanism

There are situations where users may need to use very precise criteria to filter the documents they are interested in or users may need to repeat certain queries very often. For those cases, the system provides an Advanced Search mode. To access the advanced search please click on "Show Advanced Search".

Advanced search allows you to visually create any logic expression on the different attributes related to a document. A logic expression is composed of conditions on the data attributes (e.g. "VISA result equal to ACC", "Document type different from PZ" or "Risk status equal to high risk"), and a set of three operators (AND, OR, NOT) to combine multiple conditions.

In order to create multiple conditions, users can click on + Add rule. More than one rule need to be linked through an OR operator or an AND operator. With an OR operator the expression is true if at least one of the conditions is true. With an AND operator the expression is true only if all the conditions are true. Users can select whether they want to use the OR or the AND operator by clicking the corresponding boxes. Users may also be interested in finding documents for which the expression does not hold true. In those cases, they can click in the NOT box.

In addition, different expressions with multiple conditions can also be combined to create any logical expression. To create a new expression with new conditions, click on + Add group.

In order to remove the filters on the whole documentation schedule, please click the Reset button.

Please, note that depending on the complexity of a query, its execution may take up to several minutes.

Saving search queries

In case a user wants to reuse the same advanced search several times the system allows users to save their search queries. In order to do so, please click on "Save Search" once the search expression is completed and give a name to that search expression.

The saved search expression will appear below. If you want to use the search, you just need to click on the saved search expression to re-execute it as many times as necessary.

Link to DyMaDoc

When the DYMADOC option is written in white letters with a dark grey background, users can click on it and they will be redirected to the Dymadoc page containing that particular document.

When this option is deactivated it means that the Beawre Control Risk system has not received any e-mail from Dymadoc about that particular document. Therefore, the system does not have a link to that document.

Linking documents to tasks in the project schedule

Documents can be linked to project schedule tasks by clicking on the Assign task option. A new window will unfold where tasks can be linked to documents. A document can be linked to more than one task. In such a case, the deadline for that document will be the earliest of all the deadlines established by all the tasks the document is associated to.

Delete document

In order to delete a document from the documentation schedule, please click on Delete.

Risk Analysis of a document

In order to predict the performance of a current document we learn from the historical data, looking for those documents that are similar to the current document and making a prediction based on the performance of these documents.

In particular, we provide algorithms that compare the evolution of the current document and look in historical data for documents with similar behavior. For this, the system uses a sequence matching algorithms to calculate the distance between several documents. Once distances are calculated, we use them to predict different statistics using inverse distance weighted interpolations. In order to reduce the number of documents considered and increase affinity, we first look for documents of the same subcontractor, type and phase. We also take into consideration the version number, as documents usually present different behaviors depending on whether this is the first version (1.0) of a document or a refinement of a previous version.

In particular, we predict:

  • The expected number of iterations a document will perform based on the most similar documents.
  • The dates for each remaining step and whether the last predicted step will respect the deadline established by the task the document is linked to.

Based on the predicted date, we define the level of risk based on two predefined thresholds (X and Y):

  • Low risk: the predicted date for the documents to be reviewed and approved is at least X days before the deadline. By default, X is equal to 30 days. This value can be modified if necessary.
  • Medium risk: the predicted date for the documents to be reviewed and approved is between X and Y days before the deadline. By default, Y is equal to 10 days. This value can be modified if necessary.
  • High risk: the predicted date for the documents to be reviewed and approved is later than Y days before the deadline.

When a document is not ready by the deadline required by the related task in the project schedule, the risk of not having the document ready by the deadline is not relevant anymore and we label the document as Late.

Once a document is completed, the system considers two different labels:

  • Completed: if the document finished before or on the deadline.
  • Completed Late: if the document finished after the deadline.

Exporting the documentation schedule

The system allows users to export the documentation schedule in excel format. In order to export the documentation schedule, please click on ... next to Add New Document + button. Then, click on "Save in Excel format".
